# firewiz — универсальный интерактивный менеджер firewall для Linux

**by Vladislav Pavlovich**

---

## Описание

`firewiz` — интерактивный bash-скрипт для управления всеми популярными firewall-системами в Linux из единого меню:

- **iptables**
- **ip6tables**
- **nftables**
- **ufw**
- **firewalld**

Скрипт автоматически определяет, какие системы установлены и реально активны, показывает текущее состояние каждой и даёт добавлять, удалять, блокировать, сохранять и восстанавливать правила — без необходимости помнить синтаксис каждой утилиты.

---

## Возможности

- **Автоопределение** активных firewall-систем и init-системы (systemd / sysvinit).
- **Просмотр правил** с политикой по умолчанию для цепочек INPUT / FORWARD / OUTPUT, а не только «что разрешено».
- **Добавление правил** по порту, протоколу (tcp/udp), IP/CIDR с автоматическим выбором семейства (IPv4/IPv6).
- **Удаление правил** по номеру из списка — с корректной работой для nftables (по `handle`), firewalld (по зоне и типу), ufw (по номеру) и iptables (по номеру в цепочке).
- **Быстрая блокировка адреса** одной командой во всех поддерживаемых firewall.
- **Автоматические бэкапы** перед опасными операциями (stop, reset) с ротацией — хранятся последние 20 бэкапов на каждую систему.
- **Восстановление из бэкапа** с выбором файла из списка.
- **Сброс правил** с предварительным бэкапом и подтверждением.
- **Сохранение runtime → permanent** для всех систем (iptables-save, nft ruleset dump, firewalld `--runtime-to-permanent`).
- **Валидация ввода** — портов (1–65535), IPv4, IPv6, CIDR; защита от инъекций в аргументы команд.
- **Цветной структурированный вывод**; все внешние команды вызываются через массивы аргументов (без `eval`).

## Скачивание и запуск

### Способ 1 — одной командой

```bash
curl -fsSL https://raw.githubusercontent.com/spoveddd/scripts/main/linux/firewiz/firewiz.sh | sudo bash
```

### Способ 2 — вручную

```bash
wget https://raw.githubusercontent.com/spoveddd/scripts/main/linux/firewiz/firewiz.sh
chmod +x firewiz.sh
sudo ./firewiz.sh
```

> Скрипт требует root-прав и завершится с ошибкой, если запущен от обычного пользователя.

## Бэкапы

Все бэкапы складываются в `/var/backups/firewiz/` с правами `700`. Формат имён:

| Firewall  | Файл                                  |
|-----------|---------------------------------------|
| iptables  | `iptables_YYYYMMDD_HHMMSS.rules`      |
| ip6tables | `ip6tables_YYYYMMDD_HHMMSS.rules`     |
| nftables  | `nftables_YYYYMMDD_HHMMSS.nft`        |
| ufw       | `ufw_YYYYMMDD_HHMMSS.tar.gz` (архив `/etc/ufw`) |
| firewalld | `firewalld_YYYYMMDD_HHMMSS.tar.gz` (архив `/etc/firewalld`) |

Хранится последние 20 бэкапов на каждую систему (настраивается переменной `BACKUP_KEEP` в начале скрипта). Бэкап создаётся автоматически перед `stop`, `disable` и `reset`.

---

## Сохранение правил

Пункт меню «Сохранить изменения» делает для каждой активной системы:

| Firewall  | Что выполняется                                |
|-----------|------------------------------------------------|
| iptables  | `iptables-save > /etc/iptables/rules.v4`       |
| ip6tables | `ip6tables-save > /etc/iptables/rules.v6`      |
| nftables  | дамп `nft list ruleset` → `/etc/nftables.conf` |
| ufw       | `ufw reload` (если активен)                    |
| firewalld | `firewall-cmd --runtime-to-permanent`          |

## Требования

- Linux (Debian/Ubuntu, RHEL/CentOS/Fedora, Arch и др.)
- `bash` 4+
- root-права
- Установленные утилиты тех firewall, которыми планируется управлять: `iptables`, `ip6tables`, `nft`, `ufw`, `firewall-cmd`
- Базовый набор: `awk`, `grep`, `sed`, `tar`, `tput`, `systemctl` или `service`

Если какая-то утилита не установлена — соответствующий firewall просто не появится в меню, остальные продолжат работать.

## Безопасность и ограничения

- Перед операциями, которые могут оставить систему без правил (`stop`, `disable`, `reset`), скрипт всегда создаёт бэкап.
- Все пользовательские значения (порты, IP) валидируются до подстановки в команды; внешние утилиты вызываются через массивы аргументов, без `eval`, что исключает shell-инъекции.
- Удаление правил в iptables использует нумерацию цепочки на момент вызова — если за время работы меню правила меняет другой процесс, номера могут разойтись. После удаления список перечитывается заново.
- Для nftables скрипт работает с таблицей `inet filter`, цепочкой `input`. Если её нет, она создаётся автоматически при добавлении первого правила.
- Если `iptables` использует `nf_tables` как backend (современные дистрибутивы), скрипт предупредит об этом в заголовке — правила iptables и nftables могут пересекаться.
